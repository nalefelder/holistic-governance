The Hidden Risks

Why DIY Vibe Coding Fails at Scale

Vibe coding delivers extraordinary speed — but without governance, the risks are well-documented and serious. Australia's Cyber Security Centre (ACSC) and the UK's NCSC have both flagged AI-generated code as a growing threat to organisational security.

45%

Security Vulnerabilities in AI Code

Research by Veracode (2024) found that 45% of AI-generated code introduces security flaws — including SQL injection, hardcoded credentials, and broken authentication. A separate CodeRabbit analysis of 470 GitHub pull requests found AI co-authored code has 2.74x higher vulnerability rates.

Veracode State of Software Security 2024; CodeRabbit Code Quality Report 2025
NCSC

Government Warnings on AI Code Risk

The UK National Cyber Security Centre warned in March 2026 that AI-generated code presents "intolerable risks" for many organisations. In Australia, the ACSC's Essential Eight framework and the Australian Government's Voluntary AI Safety Standard both emphasise the need for human oversight of AI-generated outputs.

UK NCSC Advisory, March 2026; Australian Cyber Security Centre; DISR Voluntary AI Safety Standard 2024
75%

Technical Debt & Maintainability

AI co-authored code has 75% more misconfigurations and 1.7x more critical issues than human-written code. Without architectural oversight, organisations inherit "comprehension debt" — code that nobody understands and that often requires a full rewrite within months.

CodeRabbit Code Quality Report 2025; Vibe Coding in Practice, arXiv:2512.11922
Privacy

Australian Privacy & IP Exposure

AI coding tools transmit source code to third-party models — potentially exposing proprietary business logic, API keys, and personal information. Under the Australian Privacy Act 1988 and the proposed reforms, organisations remain liable for data handled by AI tools. IP ownership of AI-generated code remains legally untested in Australian courts.

OAIC Australian Privacy Principles; Attorney-General's Privacy Act Review 2023; ACCC Digital Platform Services Inquiry
OWASP

Emerging AI Security Standards

The OWASP Foundation released its Top 10 Risks for AI Coding Agents in 2026, identifying critical threats including insecure code generation, prompt injection, and supply chain attacks via hallucinated package names ("slopsquatting"). These risks require mapping to existing compliance frameworks like ISO 27001 and SOC 2.

OWASP Top 10 for AI Coding Agents 2026; Checkmarx AI Security Research 2025
Real

Real-World Incidents

Documented incidents include an AI agent deleting an entire production database (Replit/SaaStr, 2025), security researchers demonstrating live data exposure in AI-built apps on BBC News (February 2026), and Wiz finding that 20% of vibe-coded applications had serious vulnerabilities or configuration errors.

Wiz Cloud Security Report 2026; BBC News, February 2026; The New Stack, March 2026
